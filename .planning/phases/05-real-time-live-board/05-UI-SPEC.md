## Component Inventory

### New Components (Phase 5)

#### 1. `LiveBoard` — `src/app/(protected)/pools/[poolId]/components/LiveBoard.tsx`

- Directive: `"use client"`
- Props: `initialPairs`, `sessionId`, `poolId`, `poolName`
- State: `pairs`, `connectionStatus` (`connected | reconnecting | disconnected`), `isRefreshing`
- Manages: Supabase Realtime subscription lifecycle (subscribe on mount, removeChannel on unmount)
- Renders: `ConnectionBanner` (conditional), `SessionBoard` (with live props), passes `refreshPairs` callback down to `AddPairForm` and `PairRow`
- Visual output: transparent wrapper — all visible UI lives in child components

#### 2. `ConnectionBanner` — `src/app/(protected)/pools/[poolId]/components/ConnectionBanner.tsx`

Position and layout:
- Sticky directly below the header (`sticky top-[57px] z-9` — header is `sticky top-0 z-10`)
- Full-width, no border-radius
- `display: none` when `status === 'connected'` (not opacity-0; eliminates layout shift)

Reconnecting state (`status === 'reconnecting'`):
- Container: `bg-slate-100 border-b border-slate-200 px-4 py-2 flex items-center gap-2`
- Spinner: `<Loader2 size={16} className="animate-spin text-slate-600" />`
- Text: `<span className="text-sm text-slate-600">Reconnecting…</span>`
- No Refresh button (auto-reconnect is in progress per D-10)

Disconnected state (`status === 'disconnected'`):
- Container: `bg-amber-50 border-b border-amber-200 px-4 flex items-center justify-between`
- Container min-height: `min-h-[44px]` (touch target for Refresh)
- Text: `<span className="text-sm text-amber-900">Disconnected — data may be stale.</span>`
- Refresh button: `<button className="text-sm font-semibold text-amber-900 underline min-h-[44px] px-2">Refresh</button>`
- Tapping Refresh triggers the same `refreshPairs()` callback used by mutations (D-09)

#### 3. `PairSkeleton` — `src/app/(protected)/pools/[poolId]/components/PairSkeleton.tsx`

Shown: during re-fetch states (e.g. post-reconnect full refresh). Not shown on initial render — SSR snapshot hydrates instantly.

Renders exactly 3 placeholder rows. Each row matches PairRow structure:

```
<div className="animate-pulse divide-y divide-slate-200">
  {[1, 2, 3].map((i) => (
    <div key={i} className="flex items-center justify-between px-4 py-3">
      <div className="h-4 bg-slate-200 rounded-md w-3/5" />
      <div className="w-10 h-10 bg-slate-200 rounded-full" />
    </div>
  ))}
</div>
```

## Interaction Contracts

### Live Count Update

- `swimmerCount` = `pairs.reduce((sum, p) => sum + p.members.length, 0)` — derived in `LiveBoard`
- `pairCount` = `pairs.length` — derived in `LiveBoard`
- Both passed as props to `SessionHeader` on every state update
- Display format: `{swimmerCount} swimmers · {pairCount} pairs` — existing format, no change

### Empty Board States (BOARD-05)

Two distinct visual states — never a blank board:

**State A — Connected and empty** (`status === 'connected' && pairs.length === 0`):
- Render the existing PairList empty state (unchanged from Phase 4):
  - Heading: `"No pairs checked in yet"` (text-base font-semibold text-slate-900)
  - Body: `"Use the form above to add the first buddy pair."` (text-base text-slate-500)
- ConnectionBanner: hidden (connected)

**State B — Disconnected** (`status === 'disconnected'`):
- Show ConnectionBanner (Disconnected variant) below header
- Show the last-known `pairs` list (stale data, not cleared)
- If `pairs.length === 0` AND disconnected: show PairList empty state AND ConnectionBanner simultaneously — counselor sees both signals

**State C — Refreshing** (`isRefreshing === true`, during re-fetch):
- Replace PairList content with `PairSkeleton` (3 placeholder rows)
- ConnectionBanner remains visible in its current state during refresh

### Subscription Lifecycle

- Subscribe: inside `useEffect`, on mount
- Channel name: `session:${sessionId}:pairs`
- Cleanup: `supabase.removeChannel(channel)` in useEffect return — prevents channel leak in React Strict Mode
- Debounce: 150ms before calling `refreshPairs` — absorbs trio INSERT burst (3 events ~5ms apart)

### Realtime → UI Status Mapping

| Supabase status | `connectionStatus` state | Banner shown |
|-----------------|--------------------------|--------------|
| `SUBSCRIBED` | `connected` | none (hidden) |
| `CHANNEL_ERROR` | `disconnected` | Disconnected |
| `TIMED_OUT` | `disconnected` | Disconnected |
| `CLOSED` | `disconnected` | Disconnected |
| (between events, auto-retry) | `reconnecting` | Reconnecting |

Note: Supabase Realtime does not emit a "RECONNECTING" status directly. Set `reconnecting` when a `CLOSED`/`CHANNEL_ERROR` is received and the channel is actively being re-subscribed (implementation detail: track via a ref or set reconnecting state after a short delay before disconnected).

## Accessibility

- `ConnectionBanner` must have `role="status"` so screen readers announce connection changes non-intrusively
- Refresh button: `aria-label="Refresh pair list"` (in addition to visible "Refresh" label)
- PairSkeleton rows: `aria-hidden="true"` (decorative loading state; screen readers should not announce placeholder rows)
- Spinner (Loader2): `aria-hidden="true"` — the "Reconnecting…" text is the accessible label
- Minimum touch targets: `min-h-[44px]` on all interactive elements (established constraint, CONTEXT.md code_context)
