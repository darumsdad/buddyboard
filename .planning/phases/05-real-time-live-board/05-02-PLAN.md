Convert the pair mutation path from `revalidatePath`-driven server refresh to client-callback-driven refresh, so that Phase 5's `LiveBoard` (not the Next.js cache) becomes the source of truth for pair data.

Output: callback-ready `AddPairForm` and `PairRow`; `actions.ts` with `revalidatePath` removed from the three pair mutations and retained in `closeSessionAction`.
</objective>
