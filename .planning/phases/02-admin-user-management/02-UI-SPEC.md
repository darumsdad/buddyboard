## Spacing Scale

Standard 8-point scale. All spacing values are multiples of 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge padding |
| sm | 8px | Compact spacing within table cells, input left padding |
| md | 16px | Default element spacing, form field gaps, table cell padding |
| lg | 24px | Section padding, modal body padding |
| xl | 32px | Layout gaps, modal width padding |
| 2xl | 48px | Major section breaks (page header to table) |
| 3xl | 64px | Page-level vertical padding (reserved, not needed this phase) |

Exceptions:
- Touch targets: `min-h-[44px]` on all interactive elements (buttons, inputs) — carried from login page (D-07, UX-01)
- Modal width: `max-w-sm` (384px) — matches login card width

## Component Inventory

### 1. Page Shell (`/admin/users`)

- Full-height white background (`bg-white min-h-screen`)
- Outer padding: `p-6` (24px on all sides)
- Max content width: `max-w-4xl mx-auto` — table readable on iPad landscape

### 2. Page Header

**Primary visual anchor:** The `<h1>` "User Management" heading (`text-3xl font-semibold text-slate-900`) and the blue-600 "Create user" button are the focal point of the `/admin/users` screen. All other elements are visually subordinate to this header row. No other element on the page competes for first visual attention.

- `<h1>` "User Management" — `text-3xl font-semibold text-slate-900`
- "Create user" button — right-aligned (`flex justify-between items-center`)
- Button: `min-h-[44px] px-4 bg-blue-600 hover:bg-blue-700 text-white text-base font-semibold rounded-md transition-colors duration-150`

### 3. Users Table

Container: `bg-white border border-slate-200 rounded-md overflow-hidden mt-8`

Header row: `bg-slate-50`

Column structure:

| Column | Width | Header copy |
|--------|-------|-------------|
| Username | flex-1 | "Username" |
| Role | 120px | "Role" |
| Created | 160px | "Created" |
| Actions | 220px | "Actions" |

Table header cell: `text-sm font-semibold text-slate-900 px-4 py-3`

Table body cell: `text-base text-slate-900 px-4 py-3`

Row divider: `divide-y divide-slate-200`

Role badge: plain text — never raw DB value. DB `"admin"` → display "Admin"; DB `"user"` → display "Counselor". (D-02)

Created column: formatted date string — `Jan 12, 2025` (no time). Use `toLocaleDateString("en-US", { month: "short", day: "numeric", year: "numeric" })`.

### 4. Row Actions

Two actions per row, in the Actions cell:

**Reset password** (secondary action):
- Trigger: text button — `text-base text-slate-600 underline-offset-2 hover:underline` — labeled "Reset password"
- Behavior: toggles an inline form below the row (local boolean state)
- No confirmation step required (D-10)

**Delete** (destructive action):
- Trigger: text button — `text-base text-red-600 underline-offset-2 hover:underline ml-4` — labeled "Delete user"
- Behavior: opens confirmation dialog (D-10)

### 5. Inline Password Reset Form

Appears below the row when triggered. Same row background (`bg-slate-50 border-t border-slate-200`).

Contents:
- Password input: `min-h-[44px] w-full border border-slate-300 rounded-md px-3 text-base text-slate-900 placeholder:text-slate-500 focus:border-blue-500 focus:ring-1 focus:ring-blue-500 focus:outline-none`
- Placeholder: "New password (min. 8 characters)"
- "Save password" button: `min-h-[44px] px-4 bg-blue-600 hover:bg-blue-700 text-white text-base font-semibold rounded-md transition-colors duration-150 disabled:opacity-50 disabled:cursor-not-allowed`
- "Discard" button: `min-h-[44px] px-4 text-base text-slate-600 hover:text-slate-900` — dismisses the inline form without saving
- Inline error below input (if validation fails)

### 6. Create User Modal

Triggered by the "Create user" button. Implemented with local boolean state — no dialog library.

**Overlay:** `fixed inset-0 bg-black/50 flex items-center justify-center z-50`

**Modal card:**
- `bg-slate-50 rounded-lg p-8 w-full max-w-sm shadow-sm` — matches login card exactly
- Title: `text-3xl font-semibold text-slate-900 text-center mb-8` — "Create user"

**Fields:**

1. Username (text input)
   - Label: `text-base font-semibold text-slate-900`
   - Input: `min-h-[44px] w-full border border-slate-300 rounded-md px-3 text-base text-slate-900 placeholder:text-slate-500 focus:border-blue-500 focus:ring-1 focus:ring-blue-500 focus:outline-none`
   - Placeholder: "Enter a username"

2. Password (password input)
   - Label: `text-base font-semibold text-slate-900`
   - Input: same classes as above
   - Placeholder: "Enter a password (min. 8 characters)"

3. Role (select dropdown)
   - Label: `text-base font-semibold text-slate-900`
   - Select: `min-h-[44px] w-full border border-slate-300 rounded-md px-3 text-base text-slate-900 focus:border-blue-500 focus:ring-1 focus:ring-blue-500 focus:outline-none bg-white`
   - Options: "Counselor" (value: `user`) and "Admin" (value: `admin`)
   - Default: "Counselor" selected

4. Submit button: "Create user" — `min-h-[44px] w-full bg-blue-600 hover:bg-blue-700 text-white text-base font-semibold rounded-md transition-colors duration-150 disabled:opacity-50 disabled:cursor-not-allowed`

5. Dismiss: text link below button — `text-base text-slate-600 hover:text-slate-900 text-center mt-2` — "Close"

### 7. Delete Confirmation Dialog

Triggered by the Delete user row action. Same overlay/card pattern as the Create User modal.

**Modal card:** `bg-white rounded-lg p-6 w-full max-w-sm shadow-sm`

**Contents:**
- Title: `text-xl font-semibold text-slate-900` — "Remove [username]?"
- Body: `text-base text-slate-700 mt-2` — "They will lose access immediately. This cannot be undone."
- Button row: `flex gap-3 mt-6`
  - "Keep user" — `min-h-[44px] flex-1 border border-slate-300 rounded-md text-base text-slate-700 font-semibold hover:bg-slate-50`
  - "Remove user" — `min-h-[44px] flex-1 bg-red-600 hover:bg-red-700 text-white text-base font-semibold rounded-md transition-colors duration-150 disabled:opacity-50`

## Interaction States

| Interaction | State | Treatment |
|-------------|-------|-----------|
| Any button (submitting) | Loading | `disabled` + `opacity-50` + `cursor-not-allowed` |
| Input (focused) | Focus | `border-blue-500 ring-1 ring-blue-500` |
| Input (validation error) | Error | `border-red-500 ring-1 ring-red-500`; inline `text-base text-red-600` message below field |
| Create modal (success) | Close | Modal closes; table reflects new user (via `revalidatePath`) |
| Password reset (success) | Collapse | Inline form collapses; no toast needed |
| Delete (success) | Row removed | Row disappears from table; no toast needed |
| Non-admin navigates to `/admin/users` | Redirect | Server redirect to `/pools` — no UI state needed |

## Accessibility

- All inputs have associated `<label>` elements via `htmlFor` / `id` pairing
- Error messages use `role="alert"` (carried from login page pattern)
- Confirmation dialog: focus trapped inside modal while open; `Escape` key closes
- Minimum touch target: `min-h-[44px]` on all buttons and inputs (D-07, UX-01)
- Color is never the sole indicator of state (error text + border both change)
