<objective>
Create the buddy call screen: a new route at `/pools/[poolId]/buddy-call` that displays a giant, high-contrast live swimmer/pair count for outdoor use (BOARD-03). The screen mirrors Phase 5's LiveBoard Realtime pattern but is a purpose-built standalone client component with its own Supabase channel. First, extract the existing `getPairsForSession` query into a shared `src/lib/pairs.ts` helper so both the existing session page and the new buddy-call page consume one implementation.

Purpose: Counselors run a verbal buddy call (count pairs aloud) and compare the spoken total against a screen they can read at arm's length in sunlight. The count must be live and the connection state must be honest (ConnectionBanner) so a stale number is never mistaken for a live one.
Output: `src/lib/pairs.ts`, new `buddy-call/page.tsx` + `BuddyCallClient.tsx` + `BuddyCallClient.test.tsx`, plus updated imports in `page.tsx` and the pairs Route Handler.
