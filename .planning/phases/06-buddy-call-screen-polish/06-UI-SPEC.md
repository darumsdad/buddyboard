## Component Inventory

### New Components (Phase 6)

#### 1. `BuddyCallClient` — `src/app/(protected)/pools/[poolId]/buddy-call/BuddyCallClient.tsx`

- Directive: `"use client"`
- Props: `initialPairs: Pair[]`, `sessionId: string`, `poolId: string`, `poolName: string`
- State: `pairs` (mirrors LiveBoard — useState initialized from initialPairs), `connectionStatus`
- Pattern: mirrors Phase 5 LiveBoard — SSR snapshot → useState → Realtime subscription
- Renders: compact header, ConnectionBanner (if not connected), giant count block, optional pair list

Layout structure (full-screen):

```
<div className="flex flex-col min-h-screen bg-white">
  {/* Compact header */}
  <header className="sticky top-0 z-10 bg-white border-b border-slate-200">
    <div className="flex items-center justify-between px-4 py-3">
      <a href={`/pools/${poolId}`}
         className="flex items-center gap-1 text-sm text-slate-600 hover:text-slate-900 min-h-[44px]">
        <ChevronLeft size={18} />
        Back to board
      </a>
      <span className="text-xl font-semibold text-slate-900">{poolName}</span>
      <div className="w-[110px]" /> {/* spacer to keep pool name centered */}
    </div>
  </header>

  {/* ConnectionBanner — sticky directly below header */}
  <ConnectionBanner status={connectionStatus} onRefresh={refreshPairs} />

  {/* Giant count — vertically centered in remaining viewport */}
  <main className="flex-1 flex flex-col items-center justify-center px-4 py-12 text-center">
    <div className="flex flex-col items-center gap-y-6">
      {/* Swimmer count block */}
      <div className="flex flex-col items-center gap-y-1">
        <p className="text-6xl md:text-8xl font-semibold text-slate-900 leading-none tabular-nums">
          {swimmerCount}
        </p>
        <p className="text-xl font-semibold text-slate-500">swimmers</p>
      </div>
      {/* Pair count block */}
      <div className="flex flex-col items-center gap-y-1">
        <p className="text-4xl md:text-6xl font-semibold text-slate-900 leading-none tabular-nums">
          {pairCount}
        </p>
        <p className="text-xl font-semibold text-slate-500">pairs</p>
      </div>
    </div>

    {/* Optional pair name list — only if pairs.length > 0 */}
    {pairs.length > 0 && (
      <ul className="mt-12 w-full max-w-sm divide-y divide-slate-200 text-left">
        {pairs.map((pair) => (
          <li key={pair.id} className="py-3 text-base text-slate-700">
            {formatPairNames(pair.members)}  {/* "First Last / First Last" */}
          </li>
        ))}
      </ul>
    )}
  </main>
</div>
```

`formatPairNames(members)` output: members joined by ` / `, each member shown as `{firstName} {lastName}`. Trios show all 3 members on one row (e.g. `Ana Ruiz / Ben Cruz / Sam Lee`) — wraps naturally.

`tabular-nums` class ensures count digits don't shift layout as numbers change.

Supabase channel name: `buddy-call:${sessionId}:pairs` — separate channel from LiveBoard's `session:${sessionId}:pairs` to allow independent lifecycle management.

## Interaction Contracts

### Buddy Call Screen — Live Count Update

- Swimmer count derived: `pairs.reduce((sum, p) => sum + p.members.length, 0)`
- Pair count derived: `pairs.length`
- Realtime subscription: subscribe to `pair` (DELETE) and `pair_member` (INSERT) on channel `buddy-call:${sessionId}:pairs` — same dual-table pattern as Phase 5 LiveBoard
- On Realtime event: debounce 150ms, then call `refreshPairs()` (GET /api/sessions/{sessionId}/pairs — reuse Phase 5 Route Handler)
- Giant count updates in place with `tabular-nums` — no animation required (counselors are reading a static number)

### Buddy Call Screen — Connection States

Identical mapping to Phase 5 (ConnectionBanner states unchanged):

| connectionStatus | Banner shown |
|-----------------|--------------|
| `connected` | none (hidden) |
| `reconnecting` | Reconnecting banner (slate) |
| `disconnected` | Disconnected banner (amber) |

If disconnected: show last-known pair count (do not reset to 0). ConnectionBanner conveys staleness.

### Buddy Call Screen — Back Navigation

- "Back to board" link: `<a href={/pools/${poolId}}>` — standard anchor, not `<Link>` router push
- ChevronLeft icon: `<ChevronLeft size={18} />` from lucide-react, inline with text, `gap-1`
- Touch target: the anchor element itself gets `min-h-[44px]` + `flex items-center` to meet the 44px minimum

### SessionHeader — Responsive Breakpoint

- Breakpoint: `md` (768px) per Tailwind default
- Below 768px: count appears in second row below pool name + Close/Logout
- At 768px and above: existing single-row layout restored
- Count format unchanged: `{swimmerCount} swimmers · {pairCount} pairs`
