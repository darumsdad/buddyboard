## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, mt-2 error messages |
| md | 16px | Default element spacing, form gap, px-4 cell padding |
| lg | 24px | Section padding, p-6 dialog content, mt-6 dialog action row |
| xl | 32px | Card/panel padding (p-8), major form group separation |
| 2xl | 48px | Empty state vertical padding (py-12) |
| 3xl | 64px | Page-level spacing (reserved for future use) |

Exceptions:

- **44px minimum touch target** — all interactive controls (buttons, inputs, icon buttons)
  use `min-h-[44px]`. Established rule from phases 1–3. Applies to: pair row Remove button,
  +1 button, Add pair button, Close session button, Log out button, pool selection buttons.
- **40px round icon buttons** — `w-10 h-10` (40px) for icon-only actions (Remove pair, +1 add
  trio). This matches the Phase 2 pattern (`DeleteConfirmDialog`, `ResetPasswordForm` icon
  buttons). These are 40px not 44px because they are supplemented by surrounding row height.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Hero count | 36px (text-4xl) | 600 (semibold) | 1 (tight) | Swimmer/pair count in session header — the primary read during buddy call |
| Heading | 20px (text-xl) | 600 (semibold) | 1.25 | Page headings (pool name in header, dialog titles) |
| Body | 16px (text-base) | 400 (regular) | 1.5 | Pair row camper names, form input text, paragraph content |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Form field labels, table column headers, sub-counts in header |

**Note (source: codebase):** Existing components consistently use `text-base` (16px) for
inputs, labels, and button text. `text-sm` (14px) appears only in table headers (CamperTable)
and secondary nav links. The hero count (36px) is new to Phase 4 — introduced for the
swimmer/pair display per CONTEXT.md D-09.

## Color

| Role | Tailwind Class | Hex Approx | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | bg-white | #ffffff | Page background, modal overlay backdrops |
| Secondary (30%) | bg-slate-50 / border-slate-200 | #f8fafc / #e2e8f0 | Card panels, form areas, table headers, sidebar |
| Accent (10%) | bg-blue-600 / hover:bg-blue-700 | #2563eb / #1d4ed8 | Reserved-for list below |

**Accent reserved for (explicit list — source: CONTEXT.md D-04, codebase patterns):**

1. "Add pair" primary action button
2. Pool selection buttons on `/pools`
3. "Join session" button in the join prompt
4. Camper chip/tag background (blue-100 bg, blue-700 text, blue-600 border) when a camper
   is resolved in the entry field
5. Input focus ring (`focus:border-blue-500 focus:ring-1 focus:ring-blue-500`)
6. "Log out" is NOT accented — use `text-slate-600 hover:text-slate-900` (secondary link
   style, matching the existing Admin nav link pattern)

**Semantic text colors:**

| Tertiary text | text-slate-500 | Placeholder text, empty state body copy |
| Error text | text-red-600 | Inline field errors (PAIR-04), server error alerts |

## Component Inventory

Phase 4 introduces the following new UI surfaces. Each maps to one or more
CONTEXT.md decisions.

### 1. Pool Selection Page — `/pools` (update existing)

**Source:** Existing `src/app/(protected)/pools/page.tsx` (replace hardcoded buttons with DB list)

- Pool buttons: same style as existing (`min-h-[44px] w-full bg-blue-600 hover:bg-blue-700
  text-white text-base font-semibold rounded-md`), driven from DB
- Admin link top-right: already implemented (`text-sm text-slate-600 hover:text-slate-900`)
- Log out button top-right (next to Admin link, same style as Admin link or a ghost button)
- Empty state: "No pools configured — contact your administrator." (text-base text-slate-500,
  centered in the card)

### 2. Join Session Prompt

**Source:** CONTEXT.md D-03, D-11 (modal, not inline banner — modal makes the shared-state
communication explicit and requires an affirmative action)

- Rendered as a centered modal overlay (`fixed inset-0 bg-black/50 flex items-center justify-center z-50`)
- Card: `bg-white rounded-lg p-8 w-full max-w-sm shadow-sm`
- Heading (text-xl, semibold): "[Pool Name] — session in progress"
- Body (text-base, slate-700): "Another counselor has an active session at this pool.
  Join to see the current pairs and add new ones."
- Primary button: "Join session" (blue-600, min-h-[44px], full width)
- Secondary button: "Go back" (border-slate-300, min-h-[44px], full width, below Join)

### 3. Session Header (sticky)

**Source:** CONTEXT.md D-08, D-09

- Position: `sticky top-0 z-10 bg-white border-b border-slate-200`
- Layout: `flex items-center justify-between px-4 py-3`
- Left: Pool name (text-xl, semibold, text-slate-900)
- Center: Hero swimmer/pair count
  - Format: "12 swimmers · 6 pairs" or "0 swimmers · 0 pairs"
  - Size: 36px (text-4xl), weight 600 (semibold), text-slate-900
  - Sub-label below (optional at mobile breakpoint): text-sm, text-slate-500
- Right: row of actions — "Close session" button + "Log out" link
  - "Close session": `min-h-[44px] px-4 border border-slate-300 rounded-md text-base
    text-slate-700 font-semibold hover:bg-slate-50` (secondary/outline style — destructive
    action revealed only in the confirm dialog)
  - "Log out": `text-sm text-slate-600 hover:text-slate-900 ml-4` (text link, no button frame)

### 4. Add Pair Form

**Source:** CONTEXT.md D-04, D-05

- Container: `bg-slate-50 rounded-md p-4` below the sticky header
- Layout: `flex flex-col gap-4` (stacked on mobile) or `flex gap-3 items-end` (row on
  md+, two CamperField columns + Add pair button)
- Labels: "Camper 1" / "Camper 2" (text-sm, semibold, text-slate-900)
- Add pair button: `min-h-[44px] px-6 bg-blue-600 hover:bg-blue-700 text-white text-base
  font-semibold rounded-md disabled:opacity-50 disabled:cursor-not-allowed`
  — disabled until both campers are resolved (D-05)
- Loading state on Add pair button: button text changes to "Adding…", opacity-50
- Server error (generic): red alert below the form (`role="alert" text-base text-red-600`)

### 5. CamperField — input with chip resolution and typeahead

**Source:** CONTEXT.md D-04, D-05; Claude's discretion for chip style

**Unresolved state:**

- Input: `min-h-[44px] w-full border border-slate-300 rounded-md px-3 text-base
  text-slate-900 placeholder:text-slate-500 focus:border-blue-500 focus:ring-1
  focus:ring-blue-500 focus:outline-none`
- Placeholder: "Code or name…"
- Typeahead fires after 300ms debounce when input length ≥ 1 character

**Resolved state (camper chip replaces input):**

- The field renders a chip instead of the text input
- Chip: `inline-flex items-center gap-1 bg-blue-100 border border-blue-300 rounded-full
  px-3 py-1 text-sm font-semibold text-blue-700`
- Chip content: "[Preferred Name] [Last Name] · [Bunk]" (matches Phase 3 D-11 format)
- Chip × clear button: `ml-1 text-blue-500 hover:text-blue-700` — clicking it clears the
  resolved camper and restores the text input

**PAIR-04 violation (source: CONTEXT.md D-07):**

- Inline error directly below the offending CamperField
- "[Camper name] is already paired in this session. Choose a different camper."
- Class: `text-sm text-red-600 mt-1 role="alert"`
- The specific field is highlighted: `border-red-400` on the chip container

### 6. Typeahead Dropdown

**Source:** CONTEXT.md D-04; Phase 3 D-10 (server-side paginated search pattern)

- Position: `absolute top-full left-0 right-0 mt-1 z-20`
- Container: `bg-white border border-slate-200 rounded-md shadow-md overflow-hidden
  max-h-48 overflow-y-auto`
- Each option: `px-3 py-2 text-base text-slate-900 hover:bg-slate-50 cursor-pointer`
- Option format: "[Preferred Name] [Last Name] · [Bunk]" — code shown if needed for disambiguation
- Keyboard nav: arrow keys + Enter to select, Escape to dismiss
- Loading row (while fetching): `px-3 py-2 text-base text-slate-500` — "Searching…"
- No results row: `px-3 py-2 text-base text-slate-500` — "No campers found"
- Debounce: 300ms (matches Phase 3 search debounce per CONTEXT.md Claude's Discretion)

### 7. Pair List

**Source:** CONTEXT.md D-08 (below Add Pair Form, scrollable)

- Container: scrollable area below the form, no explicit max-height constraint at phase 4
  (Phase 6 adds the high-contrast call screen)
- List: `divide-y divide-slate-200` rows

### 8. Pair Row

**Source:** CONTEXT.md D-08, D-06; Claude's discretion for exact layout

- Layout: `flex items-center justify-between px-4 py-3`
- Left: camper names block
  - All members listed: "[Name 1] · [Bunk] — [Name 2] · [Bunk]" (pairs)
  - Trio: "[Name 1] · [Bunk] — [Name 2] · [Bunk] — [Name 3] · [Bunk]"
  - Font: text-base, text-slate-900
- Right: action buttons row (`flex items-center gap-2`)
  - "+1" button (shown only for pairs, not trios): `w-10 h-10 rounded-full flex items-center
    justify-center text-slate-600 hover:bg-slate-100 transition-colors`
    — icon: `UserPlus` from lucide-react (size 18), title="Add third camper"
  - Remove button: `w-10 h-10 rounded-full flex items-center justify-center text-red-600
    hover:bg-red-50 transition-colors`
    — icon: `Trash2` from lucide-react (size 18), title="Remove pair"
    — matches Phase 2 DeleteConfirmDialog button style exactly

### 9. Trio Picker

**Source:** CONTEXT.md D-06

- Triggered by "+1" button on a pair row
- Rendered as a centered modal overlay (same overlay class as Join Session Prompt)
- Card: `bg-white rounded-lg p-8 w-full max-w-sm shadow-sm`
- Heading (text-xl, semibold): "Add a third camper"
- Single CamperField (same component as Add Pair Form)
- Buttons below: "Add to pair" (blue-600, min-h-[44px], full width, disabled until resolved)
  + "Keep pair as-is" (text link, text-slate-600)

### 10. Close Session Confirmation Dialog

**Source:** CONTEXT.md D-12; pattern from `DeleteConfirmDialog.tsx`

- Same modal overlay pattern as all other dialogs
- Card: `bg-white rounded-lg p-6 w-full max-w-sm shadow-sm`
- Heading (text-xl, semibold): "Close session?"
- Body (text-base, slate-700): "[N] pairs are still in the water. Close anyway? All pair
  data will be archived."
- Buttons (flex gap-3 mt-6):
  - Cancel: "Keep session open" — `min-h-[44px] flex-1 border border-slate-300 rounded-md
    text-base text-slate-700 font-semibold hover:bg-slate-50`
  - Confirm: "Close session" — `min-h-[44px] flex-1 bg-red-600 hover:bg-red-700 text-white
    text-base font-semibold rounded-md disabled:opacity-50`
- If zero active pairs: close immediately, no dialog, redirect to `/pools`

## Accessibility Contract

- All icon-only buttons carry a `title` attribute (e.g. `title="Remove pair"`, `title="Add third camper"`)
- All error messages carry `role="alert"` for screen reader announcement
- Typeahead dropdown: `aria-live="polite"` on the list container; `aria-selected` on active option
- CamperField input: `aria-label="Camper 1"` / `aria-label="Camper 2"` (label element links via htmlFor)
- Add pair button: `aria-disabled="true"` when inputs are unresolved (in addition to `disabled`)
- Focus returns to Camper 1 field after successful pair submission (D-05)
- Close session dialog: focus trap while open; focus returns to Close session button on cancel
