CamperField.tsx — "use client". This is the most complex leaf component.

Props: `{ sessionId: string; label: string; onResolved: (camper: CamperSuggestion | null) => void; error?: string | null; inputRef?: React.RefObject<HTMLInputElement> }`

State: `suggestions` (CamperSuggestion[]), `resolved` (CamperSuggestion | null), `inputValue` (string), `showDropdown` (boolean).

useTransition for isPending + startTransition. useRef for debounce timer.

Input Tailwind (from SearchBar pattern): `min-h-[44px] w-full border border-slate-300 rounded-md px-3 text-base text-slate-900 placeholder:text-slate-500 focus:border-blue-500 focus:ring-1 focus:ring-blue-500 focus:outline-none`. Add `border-red-400` to the input/chip wrapper when error prop is non-empty.

Placeholder text: "Code or name…" (from UI-SPEC Copywriting)

Debounce: on input change, clear existing timer ref, set new 300ms timer that calls startTransition(async () => { const results = await searchCampersAction(value, sessionId); setSuggestions(results); setShowDropdown(results.length > 0); }). Minimum 1 character before calling action.

When user selects a suggestion from the dropdown: set resolved=camper, call onResolved(camper), clear suggestions, setShowDropdown(false).

When the input matches an exact code (the action returns exactly 1 result from the exact-match path): auto-resolve immediately (same as selecting from dropdown).

Resolved state: render a chip `inline-flex items-center gap-1 bg-blue-100 border border-blue-300 rounded-full px-3 py-1 text-sm font-semibold text-blue-700` showing `{resolved.firstName} {resolved.lastName} · {resolved.bunk}`. Include a × clear button `ml-1 text-blue-500 hover:text-blue-700 cursor-pointer` that on click: sets resolved=null, calls onResolved(null), clears inputValue, re-focuses the input via inputRef if provided.

Typeahead dropdown: `absolute top-full left-0 right-0 mt-1 z-20 bg-white border border-slate-200 rounded-md shadow-md overflow-hidden max-h-48 overflow-y-auto`. Container must be `relative` to scope the absolute dropdown. Each option: `px-3 py-2 text-base text-slate-900 hover:bg-slate-50 cursor-pointer`. Option text format: `{m.firstName} {m.lastName} · {m.bunk}`. Loading row (isPending): `px-3 py-2 text-base text-slate-500` text "Searching…". No-results row: `px-3 py-2 text-base text-slate-500` text "No campers found". Use aria-live="polite" on dropdown container. Keyboard nav: arrow up/down to move active option (track with local state index), Enter to select, Escape to close dropdown and clear suggestions.

Error display: if error prop is truthy, render `<p role="alert" className="text-sm text-red-600 mt-1">{error}</p>` below the field container. This is where PAIR-04 violations surface (parent passes the error string from addPairAction result).
