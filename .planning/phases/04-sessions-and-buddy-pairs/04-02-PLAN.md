Implement these six items in order:

1. Private `requireAuth()` — same shape as requireAdmin in admin/users/actions.ts but WITHOUT the role check. Returns the auth session (caller can access session.user.id).

2. Private `isUniqueViolation(err: unknown): boolean` — checks `"code" in err ? err.code === "23505" : err.message.includes("23505")`. Used by addPairAction and addPairMemberAction.

3. Export `CamperSuggestion` type and `AddPairResult` discriminated union type.

4. Export `searchCampersAction(query: string, sessionId: string): Promise<CamperSuggestion[]>` — calls requireAuth; returns [] if query.trim() is empty; builds a subquery selecting camperId from pairMember where sessionId matches; first tries exact code match with notInArray filter; if none, runs ilike match on firstName, lastName, code with the same notInArray filter, limit 10.

5. Export `addPairAction(sessionId: string, poolId: string, camper1Id: string, camper2Id: string): Promise<AddPairResult>` — calls requireAuth; wraps all inserts in db.transaction: inserts one pair row with crypto.randomUUID() id, then two pairMember rows with pairId + camperId + sessionId each; catches errors and calls isUniqueViolation to distinguish PAIR-04 from other failures; on success calls revalidatePath(`/pools/${poolId}`) and returns { success: true }.

6. Export `addPairMemberAction(pairId: string, camperId: string, sessionId: string, poolId: string): Promise<AddPairResult>` — calls requireAuth; inserts a single pairMember row { pairId, camperId, sessionId }; catches isUniqueViolation; on success calls revalidatePath(`/pools/${poolId}`) and returns { success: true }. This action handles the trio (+1) use case.

7. Export `removePairAction(pairId: string, poolId: string): Promise<void>` — calls requireAuth; calls db.delete(pair).where(eq(pair.id, pairId)) — pairMember rows cascade automatically via FK; calls revalidatePath(`/pools/${poolId}`).

8. Export `closeSessionAction(sessionId: string, poolId: string): Promise<void>` — calls requireAuth; calls db.update(poolSession).set({ status: "closed", closedAt: new Date() }).where(and(eq(poolSession.id, sessionId), eq(poolSession.status, "active"))); calls revalidatePath("/pools"); calls revalidatePath("/pools/[poolId]", "page"); then calls redirect("/pools"). CRITICAL: redirect() must be called AFTER the try/catch block (not inside it) because Next.js 16 implements redirect by throwing internally — a catch block would swallow it.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Server Action | Any HTTP client can call Server Actions; auth check is the gate |
| sessionId parameter | Counselor-supplied sessionId could reference a session they don't own |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Elevation of Privilege | All exported Server Actions | mitigate | requireAuth() at top of every exported action — throws "Unauthorized" before any DB access |
| T-04-05 | Tampering | closeSessionAction sessionId parameter | mitigate | WHERE clause includes `eq(poolSession.status, "active")` — a closed session ID has no effect |
| T-04-06 | Tampering | addPairAction race condition (two devices same camper) | mitigate | DB unique constraint catches concurrent inserts; isUniqueViolation surfaces PAIR-04 error |
| T-04-07 | Information Disclosure | searchCampersAction | accept | Camper list is shared across all pools; any authenticated counselor may search all campers. No per-pool ACL needed in Phase 4. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
