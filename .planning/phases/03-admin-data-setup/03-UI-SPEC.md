## Spacing Scale

Declared values (multiples of 4). Extracted from recurring class names across login, users page, and
admin components.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | gap-1 | Sidebar nav link gap |
| sm | 8px | gap-2, mt-2 | Button-row gaps, helper text separation |
| md | 16px | p-4, gap-4, px-4 | Form field gaps, table cell padding, nav padding |
| lg | 24px | p-6, mt-6 | Page container padding, dialog button area top margin |
| xl | 32px | p-8, mt-8 | Modal content padding, section top margin |
| 2xl | 48px | — | Not currently used; available if page-level breaks needed |
| 3xl | 64px | — | Not currently used |

Exceptions:
- sm+: 12px (`px-3 / py-3`) — exception from standard set; retained because `px-3 py-3` is the established input-field and table-row padding pattern extracted from Phase 1–2 components (CreateUserModal, DeleteConfirmDialog, UserTable); changing to 8px would collapse internal padding noticeably; this exception is pre-approved.
- Touch targets: `min-h-[44px]` on all interactive elements (inputs, buttons, table action links).
  This is a hard constraint across all phases — do not reduce below 44px.
- Modal max-width: `max-w-sm` (384px) — consistent with existing CreateUserModal and DeleteConfirmDialog.

## Typography

Exactly 4 sizes, 2 weights. Extracted from existing components and matched to Tailwind defaults.

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height | Usage |
|------|------|----------------|--------|--------------|-------------|-------|
| Display | 30px | text-3xl | 600 | font-semibold | 1.2 | Page h1 headings ("Campers", "Pools") |
| Heading | 20px | text-xl | 600 | font-semibold | 1.2 | Dialog h2 headings ("Add camper", "Remove Pool?") |
| Label / Table header | 14px | text-sm | 600 | font-semibold | 1.4 | Table column headers, badge-style indicators |
| Body | 16px | text-base | 400 | (default) | 1.5 | All other text: table cells, form inputs, error messages, placeholder text |

Body labels (form field labels) use `text-base font-semibold` — body size at semibold weight.
This is the established pattern from CreateUserModal (`text-base font-semibold text-slate-900`).

Weights in use: 400 (body default) and 600 (font-semibold on headings, labels, active nav, inactive nav, buttons). No other weights are used.

## Layout Contract

### Admin Shell (applies to all admin pages in Phase 3)

The `(admin)/layout.tsx` is modified to wrap children in a flex row with the new sidebar.

```
┌──────────────────────────────────────────────────────────┐
│ AdminSidebar (w-48, shrink-0)  │  Main content (flex-1) │
│ bg-slate-50                    │  bg-white               │
│ border-r border-slate-200      │                         │
│ min-h-screen p-4               │                         │
│                                │                         │
│ [Users]                        │  <page content>         │
│ [Campers]  ← active            │                         │
│ [Pools]                        │                         │
└──────────────────────────────────────────────────────────┘
```

- Sidebar: `w-48 shrink-0 bg-slate-50 border-r border-slate-200 min-h-screen p-4`
- Active link: `block px-3 py-2 rounded-md text-base font-semibold bg-blue-600 text-white transition-colors`
- Inactive link: `block px-3 py-2 rounded-md text-base font-semibold text-slate-700 hover:bg-slate-200 transition-colors`
- Main content wrapper: `flex-1 min-w-0`
- Page container (inside main): `max-w-4xl mx-auto p-6`

Active and inactive sidebar links both use `font-semibold` (600). They are differentiated only by
background color: active uses `bg-blue-600 text-white`, inactive uses no background with
`hover:bg-slate-200`.

Source: RESEARCH.md Pattern 4 (AdminSidebar).

### Campers Page Layout

```
Page heading row:   [h1 "Campers"]  ·  [Clear all campers ← red text link]  ·  [Add camper ← blue button]
Search row:         [Search input — full width]                               [Import roster ← secondary button]
Table:              [CamperTable — see below]
Pagination:         [← Previous]  Page X of Y  [Next →]
```

Primary visual anchor: the blue "Add camper" button (blue-600, top-right of heading row).

- "Clear all campers" is a text link (`text-red-600`), not a filled button, to reduce destructive salience.
  It lives top-right next to "Add camper" to make the flow clear: both are roster-level actions.
- "Import roster" sits inline with the search bar (right-aligned) — it is a secondary action, not primary.
  Style: `min-h-[44px] px-4 border border-slate-300 rounded-md text-base text-slate-700 font-semibold hover:bg-slate-50`
- Search input: full width minus the import button. Uses the exact SearchBar pattern from RESEARCH.md Pattern 2.

### Camper Table Columns

| Column | Width | Notes |
|--------|-------|-------|
| First Name | auto | |
| Last Name | auto | |
| Code | auto | Displayed as-is (string — leading zeros preserved) |

| Actions | shrink | "Edit" (slate link) + "Remove" (red link), same pattern as UserTable |

Table container: `bg-white border border-slate-200 rounded-md overflow-hidden mt-8` (matches UserTable)
Table header row: `bg-slate-50`, cells `text-sm font-semibold text-slate-900 px-4 py-3 text-left`

### Pools Page Layout

```
Page heading row:   [h1 "Pools"]
Pool list:          [Pool Name]  [Rename]  [Remove ← red]   (one row per pool)
Add pool form:      [Pool name input]  [Add pool ← blue button]   (inline, below list)
```

Primary visual anchor: the blue "Add pool" inline form submit button (blue-600, below the pool list).

- Pool list is a simple `<ul>` — no table needed (no extra columns beyond name and actions).
- Rename action: opens an EditPoolModal (same max-w-sm modal pattern). Not inline editing.
- "Remove" is a text link (`text-red-600`) that opens DeleteConfirmDialog.
- Add pool form is inline at the bottom — no modal needed for a single text field.

## Interaction Contract

### Search (Campers)
- Input type: `type="search"` with `placeholder="Search by name or code..."`
- Debounce: 300ms (from CONTEXT.md D-10)
- Triggers: `router.push()` updating `?q=` and resetting `page=1`
- Server re-renders with filtered results
- No explicit submit button — search fires on typing after debounce

### Pagination
- Style: Prev/Next buttons flanking "Page X of Y" text
- Page size: 50 per page (from CONTEXT.md D-10)
- Prev/Next are standard `<button>` elements, disabled when on first/last page
- Disabled state: `disabled:opacity-50 disabled:cursor-not-allowed`
- Page indicator: `text-base text-slate-700`

### Modals (Add Camper, Edit Camper, Edit Pool, Import Roster)
- Backdrop: `fixed inset-0 bg-black/50 flex items-center justify-center z-50`
- Modal panel: `bg-white rounded-lg p-6 w-full max-w-sm shadow-sm` (or `bg-slate-50` for form modals)
  - AddCamperModal and EditCamperModal use `bg-slate-50` (matches CreateUserModal)
  - ImportModal uses `bg-white` (data display, not a form-only modal)
- Close: always present as a text link at the bottom: `text-base text-slate-600 hover:text-slate-900`
- Primary submit: `min-h-[44px] w-full bg-blue-600 hover:bg-blue-700 text-white text-base font-semibold rounded-md transition-colors duration-150 disabled:opacity-50 disabled:cursor-not-allowed`
- Loading state: button text changes to "Saving..." or "Importing..." with `disabled` applied

### Confirmation Dialogs (Remove Camper, Clear All Campers, Remove Pool)
- Same structure as existing DeleteConfirmDialog
- Two buttons side by side (flex gap-3): cancel (border/slate) + confirm (red/filled)
- Cancel label pattern: "Keep camper" / "Keep pool" (positive framing for cancel)
- Confirm label pattern: "Remove camper" / "Clear all campers" / "Remove pool"

### Import Modal States
1. **Default:** File input (`accept=".xlsx,.xls"`), "Download sample template" link, "Upload roster" button
2. **Pending:** "Uploading..." button text, disabled, no spinner (consistent with existing loading states)
3. **Error:** Unordered list of row-level errors below the file input, styled `text-sm text-red-600`, wrapped in `role="alert"`
4. **Success:** "Imported N campers successfully." in `text-sm text-green-700`, modal offers "Close" link

Sample template link: `<a href="/sample-roster.xlsx" download>` — styled as `text-base text-blue-600 underline-offset-2 hover:underline`

### Form Validation (Add/Edit Camper, Add Pool)
- Required fields: validated server-side via Server Action. No client-side custom validation beyond `required` attribute.
- Error display: single `<p role="alert" className="text-base text-red-600">` paragraph below the last field, above the submit button.
