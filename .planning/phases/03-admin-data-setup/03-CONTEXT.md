# Phase 3: Admin Data Setup - Context

**Gathered:** 2026-06-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Admin can configure pools (add, rename, remove without touching code, 3 defaults on fresh install) and manage the full camper roster — bulk import via Excel, individual CRUD (add/edit/remove), and search/filter. Admin also has a "Clear all campers" action for season rollover. Counselor-facing screens (buddy pair entry, typeahead) do NOT exist in this phase — those are Phase 4.

**Success criteria (from ROADMAP.md):**
1. Admin can upload an Excel spreadsheet and see all campers imported with codes stored and displayed as strings (leading zeros preserved)
2. Admin can add, edit name or code, and remove individual campers from the roster
3. Admin can search and filter the camper list by name or code and see results update immediately
4. Admin can add, rename, and remove pools from the admin screen without touching any code or config files
5. On a fresh install, three pools exist in the system by default

</domain>

<decisions>
## Implementation Decisions

### Admin Navigation
- **D-01:** Claude's discretion — use a left sidebar for admin navigation. This is the conventional admin panel pattern and aligns with admin tasks being desktop-primary. Phase 2 explicitly deferred this decision here. The sidebar links: Users / Campers / Pools. Lives in the `(admin)` layout so it appears on every admin screen.

### Pool Configuration
- **D-02:** Pools are stored in a new `pool` table in the database (not config files). This makes POOL-01 (add/rename/remove from the admin screen) straightforward CRUD with no code changes.
- **D-03:** Default pool names are "Pool 1", "Pool 2", "Pool 3" — admin renames them from the UI before the season. Seeded via a Drizzle DB migration (same pattern as the admin user seed in Phase 1). Migration inserts the 3 defaults only on initial schema creation.

### Camper Roster — Excel Import
- **D-04:** Import is destructive replace — uploading a spreadsheet replaces the entire camper roster. Simple mental model: the spreadsheet IS the roster. No merge or append logic.
- **D-05:** Import fails all-or-nothing: if any row has a problem (missing column, blank code, duplicate code within the sheet), nothing is imported and the admin sees row-level error details ("Row 12: code is blank").
- **D-06:** Required Excel columns: First Name, Last Name, Code. Optional columns: Bunk, Notes. Codes are stored as strings (CAMP-02 — leading zeros preserved). Bunk and Notes are informational only — stored in the camper record and visible to admins, never surfaced in counselor buddy pair entry or the buddy board.
- **D-07:** A downloadable sample spreadsheet template is generated by the planner so admins know the expected format before uploading.

### Camper Roster — CRUD & Search
- **D-08:** Add individual camper form collects: First Name, Last Name, Code (required), Bunk (optional), Notes (optional).
- **D-09:** "Clear all campers" action removes the entire roster in one operation. Required for season rollover. Show a confirmation dialog before executing.
- **D-10:** Camper search is server-side with pagination — ~1,000 campers on the roster. Admin types in a search box (debounced), request hits the server, results update. Paginate the list (50 per page). Satisfies CAMP-06 "results update immediately" at scale.

### Claude's Discretion
- Left sidebar design details (width, active state style, icons vs. text-only) — follow established Tailwind patterns (slate palette, blue-600 accent).
- Debounce timing for search (300ms is standard).
- Exact pagination UI (numbered pages vs. prev/next controls).
- Edit flow for individual campers (inline row editing vs. modal) — modal is conventional and consistent with Phase 2's user management patterns.
- Error display style for Excel row errors — inline list below the upload area.

</decisions>

<specifics>
## Specific Ideas

- Excel import: destructive replace with all-or-nothing validation. Row-level errors displayed inline below the upload area (e.g., "Row 12: code is blank", "Row 7: duplicate code '042' found in rows 7 and 23").
- Bunk and Notes columns: optional in Excel and in the manual add form. Visible in the admin camper list as additional columns. Stored but not indexed or searched against.
- Default pool names: "Pool 1", "Pool 2", "Pool 3" — admin renames from the pools admin screen before the season.
- "Clear all campers" is a top-of-page destructive action with a confirmation dialog similar to Phase 2's delete user dialog.
- Camper search: server-side, debounced (~300ms), paginated (50 per page). Search matches against first name, last name, and code fields.

</specifics>
