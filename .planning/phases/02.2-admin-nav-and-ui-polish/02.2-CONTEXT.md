# Phase 2.2: Admin Nav & UI Polish - Context

**Gathered:** 2026-06-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Three tightly-coupled UI improvements that make the admin experience coherent:
1. Role-based post-login redirect — admins land on `/admin/users`, counselors land on `/pools`
2. Bidirectional nav links — admin screen gets a "Buddy Board" link top-right; pools screen gets an "Admin" link top-right (admin-role only)
3. Round icon buttons — replace text-link-styled "Reset password" / "Delete user" buttons in the UserTable actions column with round icon buttons (lucide-react icons, native title tooltips)

</domain>

<decisions>
## Implementation Decisions

### Post-Login Redirect
- **D-01:** Role detection happens in `src/app/(auth)/login/page.tsx` — read `data.user.role` from the `authClient.signIn.username()` response and push to `/admin/users` if `admin`, otherwise push to `/pools`.
- **D-02:** No new files or root-page router needed — minimal change to the existing login page.

### Navigation Links
- **D-03:** Admin screen (`/admin/users`): "View Buddy Board" plain text link in the top-right of the page header row, inline with the "User Management" h1 on the left.
- **D-04:** Pools screen (`/pools`): "Admin" plain text link in the top-right corner of the page, conditionally rendered only when the current user has the `admin` role.
- **D-05:** Link style: `text-sm text-slate-600 hover:text-slate-900` — subtle, does not compete with primary action buttons.

### Round Icon Buttons
- **D-06:** Install `lucide-react`. Replace text-link buttons in the UserTable actions column with round icon buttons.
- **D-07:** Icon pair: `Key` icon for "Reset password", `Trash2` icon for "Delete user".
- **D-08:** Add `title="Reset password"` and `title="Delete user"` attributes for native browser tooltip discoverability.
- **D-09:** Button style: round (`rounded-full`), 40×40px (`w-10 h-10`), centered icon, `text-slate-600 hover:bg-slate-100` for reset, `text-red-600 hover:bg-red-50` for delete. Meets `min-h-[44px]` touch target via the existing cell padding context.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `authClient.signIn.username()` — already returns user role in response data; no extra API call needed for redirect
- `auth.api.getSession({ headers })` — server-side session read; already used in `pools/page.tsx` and `(admin)/layout.tsx`; use same pattern to conditionally render Admin link on pools page
- `DeleteConfirmDialog` and `ResetPasswordForm` — both have an `open` state toggled by a trigger button; only the trigger button's JSX needs to change

### Established Patterns
- Server components read session via `auth.api.getSession({ headers: await headers() })` — follow this for role check on pools page
- `min-h-[44px]` on all interactive elements — round buttons meet this via table cell padding; confirm `w-10 h-10` (40px) is acceptable given surrounding padding
- `text-slate-600` for secondary text, `red-600` for destructive — carry into icon color tokens

### Integration Points
- Login page → role-based push replaces static `/pools` push
- UserTable actions column → `ResetPasswordForm` and `DeleteConfirmDialog` trigger buttons restyled in place
- Pools page → session role check added at top; Admin link conditionally rendered

</code_context>
