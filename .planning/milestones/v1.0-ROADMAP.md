## Overview

BuddyBoard v1.0 delivers a complete, production-ready pool safety tracking app for Camp Regesh. Counselors register buddy pairs at the pool using typeahead or camper codes, see live updates on every device simultaneously via Supabase Realtime, and run buddy calls from a high-contrast outdoor-optimized screen. Admins manage counselor accounts, import the full camper roster from Excel, and configure pools — all without touching code.

## Milestone Summary

**Decimal Phases:**
- Phase 2.1: User Name Fields (inserted after Phase 2 — real names needed for user table UX)
- Phase 2.2: Admin Nav & UI Polish (inserted after Phase 2 — role-based redirect and bidirectional nav)

**Key Decisions:**
- Better Auth admin plugin used for user management — no custom auth logic
- Supabase Realtime chosen for multi-device sync — avoids custom WebSocket infrastructure
- Subscribe to BOTH pair_member (INSERT) AND pair (DELETE) — cascade-deleted pair_member rows don't fire Realtime events
- revalidatePath removed from pair mutation actions — Realtime subscription handles UI refresh
- Camper codes stored as text throughout — never cast to number to preserve leading zeros
- Session table name is poolSession (Drizzle) / pool_session (DB) — `session` owned by Better Auth
- Hosted on Vercel (switched from initial Hostinger plan — Vercel matched stack naturally)
- Supabase Row Level Security skipped for v1 — auth enforced at Next.js middleware layer

**Issues Resolved:**
- Trio INSERT compatibility with Supabase Realtime: INSERT into pairMember (not UPDATE) preserves event stream
- useEffect cleanup: supabase.removeChannel(channel), never channel.unsubscribe() (prevents TooManyChannels)
- Next.js 16 redirect() must be called outside try/catch in Server Actions
- Next.js 16 params and searchParams must be awaited before property access

**Technical Debt Incurred:**
- Pre-existing test failures in admin action tests (DB ECONNREFUSED in test env) — deferred to v1.1
- Supabase Row Level Security not implemented — next milestone security hardening
- No email-based password reset (admin resets manually) — accepted for v1, deferred
