# BuddyBoard — v1.0 Requirements Archive

**Archived:** 2026-06-28 at v1.0 milestone close
**Status:** All requirements complete

---

## v1 Requirements

### Authentication (AUTH)

- [x] **AUTH-01**: User can log in with username and password — ✓ Phase 1
- [x] **AUTH-02**: Authenticated session persists across browser restarts (remember me) — ✓ Phase 1
- [x] **AUTH-03**: Admin can add a new user account (username + password + role + first/last name) — ✓ Phase 2 + 2.1
- [x] **AUTH-04**: Admin can remove a user account — ✓ Phase 2
- [x] **AUTH-05**: Admin can reset any user's password from the admin screen (no email flow) — ✓ Phase 2

### Camper Roster (CAMP)

- [x] **CAMP-01**: Admin can upload an Excel spreadsheet to bulk-import campers (name + admin-assigned code) — ✓ Phase 3
- [x] **CAMP-02**: Camper codes are stored and treated as strings (preserves leading zeros) — ✓ Phase 3
- [x] **CAMP-03**: Admin can add an individual camper manually (name + code) — ✓ Phase 3
- [x] **CAMP-04**: Admin can edit a camper's name or code — ✓ Phase 3
- [x] **CAMP-05**: Admin can remove a camper from the roster — ✓ Phase 3
- [x] **CAMP-06**: Admin can search and filter the camper list by name or code — ✓ Phase 3

### Pool Configuration (POOL)

- [x] **POOL-01**: Admin can add, rename, and remove pools without a code change — ✓ Phase 3
- [x] **POOL-02**: System ships with 3 pools configured by default — ✓ Phase 3

### Sessions (SESS)

- [x] **SESS-01**: Counselor can start a new pool session by selecting a pool from the configured list — ✓ Phase 4
- [x] **SESS-02**: System prompts for confirmation before closing a session that has active pairs — ✓ Phase 4
- [x] **SESS-03**: Session data (buddy pairs, timestamps) is archived when a session is closed — not deleted — ✓ Phase 4
- [x] **SESS-04**: Closed sessions are not shown in the active pool view — ✓ Phase 4

### Buddy Pairs (PAIR)

- [x] **PAIR-01**: Counselor can register a buddy pair by entering two camper codes — ✓ Phase 4
- [x] **PAIR-02**: Counselor can register a buddy pair by typing a name with live typeahead search — ✓ Phase 4
- [x] **PAIR-03**: Counselor can remove a buddy pair; removal is confirmed by server before the UI updates — ✓ Phase 4
- [x] **PAIR-04**: System prevents the same camper from appearing in two active pairs simultaneously (enforced at the database level) — ✓ Phase 4
- [x] **PAIR-05**: All buddy pair changes (add, remove) appear in real time on all devices connected to the same pool — ✓ Phase 5

### Buddy Board (BOARD)

- [x] **BOARD-01**: Counselor can view a live, scrollable buddy board showing all active pairs for their pool — ✓ Phase 5
- [x] **BOARD-02**: Buddy board displays total swimmer count and pair count (e.g. "12 swimmers — 6 pairs") — ✓ Phase 5
- [x] **BOARD-03**: Counselor can open a buddy call screen — a clean, high-contrast, scrollable list of all pairs optimized for outdoor use (large text, high contrast, large touch targets) — ✓ Phase 6
- [x] **BOARD-04**: App displays a prominent connection status indicator ("Connected / Reconnecting / Disconnected — data may be stale") — ✓ Phase 5
- [x] **BOARD-05**: Empty buddy board clearly communicates why it is empty (no pairs checked in vs. not connected / loading) — ✓ Phase 5

### Devices (UX)

- [x] **UX-01**: App is fully usable on iPad and laptop (primary experience) — ✓ Phase 6
- [x] **UX-02**: App is best-effort responsive on iPhone — all features accessible, layout may be compact — ✓ Phase 6

## v2 Requirements (Deferred)

- Buddy call check-off / confirmation tracking per pair
- Email-based password reset
- Viewing or searching session history / past buddy boards
- Bulk remove all pairs (end-of-session clear)
- Counselor-facing notifications (e.g. alert when a pair has been in the pool unusually long)
- Print-friendly buddy call sheet

---

## Out of Scope

- **OAuth / third-party login** — unnecessary complexity for camp staff
- **Pool transfer** — kids re-register fresh at the new pool; no pair migration between pools
- **RFID / wearable / NFC integration** — out of scope for v1
- **Parent-facing features** — no parent portal or notifications
- **Native mobile app** — responsive web covers the iPhone use case
- **Swimming zone / depth tracking** — physical safety infrastructure, not this app's concern
- **Automated session scheduling** — counselors open and close sessions manually
